## 一、一句话定位

> **Home** 是一款让主人用最简单的方式"装修"出自己风格的家名片，作为朋友做客的预热指南。
>
> 灵感来自极简酒水电子菜单——把"调酒师的作品"变成"主人的家"。

---

## 二、产品哲学（7 条原则）

| # | 原则 | 含义 |
|---|------|------|
| 1 | **3 分钟出第一张作品** | 新用户 3 分钟内必须能产出第一张可分享的卡片（含教学关卡中选定的那张） |
| 2 | **不会出废品** | 基础模板有"最低审美保证" |
| 3 | **AI 是放大器** | 用户自己能玩，AI 是给"想更好"的人加 buff |
| 4 | **想象叠加层** | 在真实空间上叠加主人的想象 |
| 5 | **真实 + 涂鸦** | 核心内容形态 = 真实照片 + 手绘涂鸦动画 |
| 6 | **选 > 写** | 默认行为是选，不是写 |
| 7 | **卡牌化 + 非游戏化** | 有玩感，无机制（无等级 / 积分 / 排行榜） |

### 次用户（接收者 / 客人）
- 收到 home 链接的朋友
- 通过 home 提前了解主人家 + 提需求

## 七、关键用户流程

### A. 创建 Home（主人）
```
1. 进入 → AI 引导师打招呼（引导师是有自己 home 的独立角色）
2. 引导师提供两个选择：
   ├── "我想直接开始"       → 跳到步骤 4
   └── "先看看你的 home"    → 进入教学关卡（步骤 3）
3. 教学关卡（任天堂式，极短）：
   - 引导师展示自己正在装修的 home
   - 引导师导入一张照片（含小动物）→ 问"这里面有什么你在意的吗？"
   - 用户输入（如"一只小猫"）→ AI 生成多张调性卡牌
   - 用户选一张 → "太好了！你刚帮我的小猫做了一张卡片！"
   - 引导师邀请用户手绘一个简单贴纸 → 选择动效 → 贴到引导师的 home 上
   - 系统记住用户选择的调性，后续创建时优先推荐（灵感种子）
   - 贴纸存入用户的贴纸库，可在自己的 home 中使用
   - 其他教学内容留在引导师的 home，不可带走
   → 进入步骤 4
4. AI 引导师以邀请式展示判断风格倾向
   - 引导师展示 2-3 组场景卡片（不同氛围的空间图片）
   - 邀请用户点选"你更喜欢哪个感觉？"
   - 用户选择 → AI 暗中确定材质调性（绒毛/石质/叶脉/木质）
   - 全程是邀请而非提问，符合引导师服务型性格
5. 创建房间卡：上传照片 → 选/画/AI 生成涂鸦 → 写描述 → AI 生成润色卡牌 → 选一张或保留原话
6. 创建成员卡：选关系 → 写大白话 → AI 生成润色卡牌 → 选一张或保留原话
7. 写使用说明 → AI 生成润色卡牌 → 选一张或保留原话
8. 预览 → 一键发布 → 分享链接

注：每张卡的文字调性可独立选择（通过 AI 卡牌选择机制），
    但视觉风格（简笔画）锁定为步骤 4 确定的材质，保证审美一致性。
```

### B. 接收 Home（客人）
```
1. 收到链接 → 开门入户仪式（信封/信纸）
2. 浏览：房间 / 成员 / 使用说明 / 菜单
   → 浏览过程中，遇到主人预设的问卷互动点
   → 例：菜单卡上嵌入"你有什么忌口？"，活动卡上嵌入"你什么时间方便？"
3. 到访前看 home（建立"赛博层"印象）
4. 到访时凭记忆感受（home 是预热，不是现场指南）
5. 做客后留印象笔记 + 在足迹地留言
```

### C. 异步社交（足迹地）
```
客人 A 来访 → 在 home 某位置留言"这杯威士忌真的绝"
   ↓
所有人可见（主人和其他客人都能看到）
   ↓
客人 B 来访 → 看到 A 的留言 → 在同一位置也留言 → 留言叠加显示
   ↓
客人可管理自己的留言（删除 / 保留）
主人可管理 home 内所有留言（删除 / 隐藏）
```

## 八、视觉与动效规范

### 8.1 视觉风格
- **基础风格**：日系简笔画（手绘线条）
- **UI 基底**：真实纸张纹路（纸质纤维感，整个产品在同一个材质语言中）
- **配色**：低饱和度 + 米色为底，温和色系
- **线条**：默认用墨黑色线条（浅色底）或白色线条（深色底）；部分材质调性（叶脉/漫画/西幻）使用彩色
- **字体**：圆体 / 少棱角字体（中文）+ 等线体（英文）
- **标签设计**：手绘线条 + 纸张压印 → 拟真感
- **留白**：大量留白，真实材质结合简笔画
- **整体感觉**：细腻柔和，略带卡通感，像一张真正的手作邀请函

### 8.2 UI 动效规范（产品交互层）
- **卡牌入场**：从画面外滑入 + 轻微弹落（纸张摩擦感）
- **卡牌选中**：浮起（阴影加深，像从桌面拿起）
- **AI 润色卡出现**：从用户原文卡底部"抽出来"
- **确认选择**：轻按落定（像盖章）
- **页面过渡**：翻纸 / 翻书效果
- **开门入户**：信封出现 → 信封打开 → 信纸抽出 → 展开内容
- **文字出现**：普通显现（已是写好的信，不做墨迹效果）
- **节奏**：所有动效 ease-out，细腻温和，不夸张
- **纸质拟真**：UI 基底是纸，动效也应像纸（翻开、挪开、滑入）

### 8.3 涂鸦动效规范（想象叠加层）
- **放置时**：用户自选动效类型
- **预设动效集**：
  - 线条抖动 / 旋转 / 整体扭动 / 左右跳跃 / 上下跳跃
  - 呼吸（微微放大缩小）/ 描线（线条循环重描）
  - 飘落（轻微下坠弹回）/ 眨眼（角色/宠物类）
- **原则**：不夸张，细腻温和，让涂鸦"活"起来
- **材质影响动效节奏**：⏳ 实践中验证（绒毛更软更慢，石质更精确克制，叶脉更不规则，木质更稳缓）

### 8.4 涂鸦风格
- **笔触**：1-2px 细线 + 偶尔粗笔强调
- **颜色规则**：
  - **手动绘画**：RGB 色相任选，不受材质限制
  - **AI 生成 · 绒毛/石质/木质**：单色风格（深底白线 / 浅底黑线）
  - **AI 生成 · 叶脉**：彩色（活泼、有装饰性小元素）
  - **AI 生成 · 漫画**：彩色（速度线、网点、漫画分镜感）
  - **AI 生成 · 西幻**：带色彩（纹章、符文装饰）
- **瑕疵保留**：故意保留"手绘感"（不完美 = 真实）

## 九、AI 调性指南

### 日常调性（4 种材质）

#### 🧶 绒毛系
- **文字调性**：柔软、温暖、慢生活、想靠近
- **简笔画风格**：圆润柔软的线条，笔触略带毛边感，像用蜡笔或软铅笔画出。线条末端自然收圆，无锐角。
- **适用场景**：宠物 / 温馨家居 / 慢节奏的生活细节

#### 🪨 石质系
- **文字调性**：克制、干净、留白、不解释
- **简笔画风格**：精确均匀的细线条，几何感强，粗细一致。像用针管笔画出，干净利落，几乎没有多余笔触。
- **适用场景**：现代空间 / 极简主义 / 少即是多的态度

#### 🌿 叶脉系
- **文字调性**：有梗、灵动、网感、不按常理
- **简笔画风格**：不规则的活泼线条，粗细变化大，偶尔出现夸张的弧线和小装饰（星星、波浪线、感叹号）。像随手在笔记本边角画的涂鸦。**使用彩色**以保持活泼感。
- **适用场景**：年轻人的家 / 合租 / 宠物 / 一切有幽默感的日常

#### 🪵 木质系
- **文字调性**：沉稳、有故事、自然、文人气
- **简笔画风格**：有笔锋的线条，粗细变化如水墨白描。线条流畅而有节制，带中式绘画的韵味，像用毛笔画出的速写。
- **适用场景**：中式空间 / 老房子 / 有岁月感的人和物

### 扩充单元（额外可选风格）

#### 🐉 西幻系
- **文字调性**：史诗感、古老、神秘、中世纪
- **简笔画风格**：装饰性线条，带有纹章和卷轴感。线条较复杂，可能出现花边框、符文元素，像手抄本里的插图。**带色彩**以呈现装饰感。
- **适用场景**：游戏 / 二次元 / 喜欢用史诗感描述日常的人

#### 📔 漫画系
- **文字调性**：热血、夸张、分镜感、表情丰富
- **简笔画风格**：粗犷有力的轮廓线，可能出现速度线、集中线、网点效果。像漫画分镜里的角色和物件。**使用彩色**以呈现漫画分镜感。
- **适用场景**：二次元 / 漫画爱好者 / 想让 home 有漫画分镜感的人

### 调性交互机制
- **调性 = 文字风格 + 简笔画风格**，打包不可拆
- 视觉风格（简笔画）由 AI 根据问答确定材质后锁定，全 home 统一，不可更改（保证审美下限）
- 文字调性每张卡可独立选择（通过 AI 卡牌选择机制）
- 用户写大白话 → 原文直接成为卡片 → AI 分析感受性词汇 → 生成 1-2 张最匹配调性的润色卡牌 → 半露于原文卡下方
- 用户可展开查看 AI 卡牌，选择一张或保留原话
- AI 润色是"平行的可能性"，不是"更好的版本"

## 十、技术栈与平台建议

### 10.1 平台策略
- **MVP**：Web 单端（D1-1）
- **V2**：加微信小程序（社交裂变）
- **V3**：App（性能 + 体验）

### 10.2 推荐技术栈
| 层 | 推荐 | 理由 |
|---|------|------|
| 前端框架 | **Next.js 15** | App Router + React 19 + PPR |
| UI 库 | **Tailwind CSS + shadcn/ui** | 快、精致、可定制 |
| 动画 | **Motion (motion/react) + Lottie** | Framer Motion 继任者，更小更快 |
| 画板 | **Konva.js (react-konva)** | Layer 层级天然映射照片+涂鸦叠加 |
| 后端 | **Supabase** | Auth + DB + Storage 一体 |
| 纸质效果 | **SVG Filter** | feTurbulence 纸张纤维 + mix-blend-mode 墨纸混合 |
| AI 编排 | **Vercel AI SDK 4.x** | 多 provider 统一接口 + 流式响应 |
| 部署 | **Vercel + Cloudflare R2** | 全球 CDN |

### 10.3 AI 集成
| 功能 | 推荐方案 | 说明 |
|------|----------|------|
| AI 引导师 | **Claude Sonnet** (via AI SDK) | 流式对话，理解力强 |
| 文字润色 | **Gemini 2.5 Flash** | $0.15/$0.60 per 1M token，中文能力强，1M 上下文（fallback: GPT-4.1 mini） |
| 涂鸦生成 | **Flux.1 Schnell** (via fal.ai) | ~$0.003/张，1-4步出图，Apache 2.0 |
| 涂鸦叠加 | **Flux + ControlNet** (lineart/scribble) | 照片→涂鸦风格叠加 |
| 涂鸦美化 | ControlNet + 材质风格 prompt | 按材质调性统一涂鸦风格 |

### 10.4 AI 调用成本估算
| 操作 | 模型 | 单次成本估算 |
|------|------|-------------|
| AI 引导师对话 | Claude Sonnet 4 | ~$0.003-0.015/次 |
| 文字润色（1-2张卡） | Gemini 2.5 Flash | ~$0.00015-0.0006/次 |
| 涂鸦生成 | Flux Schnell | ~$0.003/张 |
| 单个 home 全流程 | 混合 | ~$0.10-0.30/home |

### 10.5 画板系统实现
```
[静态画板]
react-konva (Konva.js) → 用户在照片上绘画 → 直接叠加

[贴纸画板]
react-konva (Konva.js) → 用户绘画 → 导出 PNG → 选择动效 → 存入贴纸库

[Home来画 · 纯语言]
用户文字描述 → fal.ai (Flux Schnell + ControlNet + 材质 prompt) → 输出 PNG → 叠加到画板

[Home来画 · 画+语言]
用户手绘草图 + 文字描述 → Flux + ControlNet (scribble condition) → 以草图为参考重绘 → 输出 PNG
```

| 技术 | 阶段 | 用途 |

| **View Transitions API** | M1 | 页面过渡（翻纸效果），替代部分 Motion，更轻量 |
| **CSS Anchor Positioning** | M2 | 引导师定位、涂鸦标签定位 |
| **Supabase Realtime** | M3 | 足迹地实时留言叠加显示 |
| **Cloudflare Workers AI** | M2 评估 | 边缘生成涂鸦，可能替代 fal.ai 降低延迟至 1-3 秒 |
| **WebGPU 涂鸦动效** | V2 | GPU shader 渲染涂鸦动效，替代 Lottie |

### 10.8 性能优化策略
| 优化项 | 方案 |
|--------|------|
| **纸张纹理** | 预渲染 2-3 种静态纹理 PNG，按材质调性选用，不做实时 SVG filter |
| **涂鸦动效** | 常用动效预导出为 Lottie JSON，比实时计算更流畅 |
| **图片加载** | 低分辨率模糊占位图（blur placeholder）→ 渐进加载高清图 |
| **开门入户动画** | 预渲染为 Lottie，避免首屏实时计算 |
| **AI 涂鸦生成** | loading 动画：调用用户此前创作的任意一张贴纸 + 随机动效 |

## 十一、关键模块详细设计

### 11.1 AI 引导师 + 开场仪式
#### 引导师形态
- **形态**：一条简约圆润的墨黑色线条，与涂鸦视觉语言一致
- **长度**：自由伸缩——短如标点，长可引导视线、圈出区域
- **功能性**：可形成符号（?、箭头等）、圈出区域、引导视线、偶尔表情

#### 引导师性格
- **存在**：赛博生命，在 Home 中装修了自己的赛博家（教学关卡场景）
- **性格**：温和、有距离感、安全
- **态度**：服务型——需要时在，不需要时安静
- **口吻**：邀请式——"这是我的家，你想看看吗"
- **原则**：不主动提问、不建议、不评判用户选择
- **服务方式**：不问"要不要帮忙"，直接将结果以卡牌形式呈现，用户选或不选

#### 命名
- 引导师说"你好，你可以叫我 h____"，用户自行填写（h 开头的任意词）

#### 开场仪式（首次创建 home）
- 画面中央：一笔画线条构成的半开的门（静止）
- 用户点击门
- 线条流动、脱离门的形态，汇聚成一团
- 线条展开，形成文字："欢迎来到，home。"
- 文字消散，线条恢复为自由形态
- "你好，你可以叫我 h____"
- → 引导师正式就位，进入教学/创建选择
- **含义**：门 = 引导师 = Home 的视觉语言，用户亲手打开了这个赛博生命

#### 教学关卡
- 引导师展示自己温暖但不完美的赛博家
- 故意留未完成感让用户参与
- 用户在引导师的照片上完成一次成员卡创建（选卡牌）
- 用户手绘一个简单贴纸 + 选择动效 → 贴到引导师的 home 上（动态贴纸教学）
- 系统记住用户选择的调性作为"灵感种子"，后续创建时优先推荐
- 用户手绘的动态贴纸存入用户的贴纸库
- 其他教学内容留在引导师的 home，不可带走

#### 技术实现
- 开场一笔画变形动画 → Lottie 预渲染（整个产品最有记忆点的动画，需要精细调校）
- 常见形态（问号、圆圈、箭头）→ Lottie 预渲染
- 动态行为（伸缩、浮动、跟随）→ SVG path 实时计算

### 11.2 开门入户（访客视角）
- **目标**：让客人进入 home 时有"收到邀请函"的仪式感
- **形态**：信封出现 → 信封打开 → 信纸抽出 → 展开内容（纸质拟真动效）
- **要素**：手绘线条动画 + 主人问候语 + 背景音乐（可选）
- **MVP**：3-5 种信封/信纸模板可选
- **注**：引导师的开场仪式仅发生在用户首次创建 home 时；访客看到的是主人选择的开门模板

### 11.3 房间卡
- **结构**：1 张照片 + 想象叠加层 + 使用说明
- **照片**：用户上传，后端自动压缩（最大 10MB，压缩至适配分辨率）
- **数量**：房间卡数量不限，首屏加载前 3 张，其余懒加载
- **想象叠加层**：手绘线条涂鸦（持续动效），风格由材质调性决定
- **使用说明**：用户写大白话 → 原文成卡 → AI 生成润色卡牌 → 选择或保留原话

### 11.4 成员卡
- **类型**：主人 / 室友 / 宠物
- **结构**：头像 + 名字 + 性格描述
- **宠物特别支持**：用宠物视角自述（"我，XX 喵，活了 300 年"）
- **AI 润色**：每张卡独立选择文字调性（用户也可保留自己的原话）

### 11.5 使用说明
- **场景**：物品 / 动物 / 空间都有"性格"
- **AI 润色**：用户写大白话 → AI 按材质调性生成润色卡牌 → 选择或保留原话

### 11.6 温柔预热（问卷）
- **形态**：嵌入在特定卡片上的互动点（非独立环节）
- **位置**：由主人决定放置在哪些卡片上（如菜单卡、活动卡）
- **问卷类型**：
  - 忌口 / 过敏
  - 偏好（口味 / 兴趣）
  - 同行人数 / 抵达时间
  - 特殊需求
- **触发**：客人浏览到对应卡片时自然弹出
- **数据**：只有主人家可见
- **MVP**：3-5 个标准问题模板，主人选择后放置在卡片上

### 11.7 足迹地（叠层留言）
- **形态**：home 内嵌"留言板"
- **位置**：主人可指定 home 内的"留言地"（如：厨房、客厅、入口）
- **留言形式**：手写涂鸦风短句
- **叠层**：多个客人留言叠加显示
- **可见性**：所有人可见（主人和所有客人都能看到）
- **权限**：
  - 客人：管理自己的留言（删除 / 保留）
  - 主人：管理 home 内所有留言（删除 / 隐藏）
- **灵感**：艾尔登法环 / 黑暗之魂异步留言

### 11.8 编辑与发布
- **发布后可修改**：主人可以随时编辑 home 内容
- **链接即快照**：每次分享链接记录发送时刻的 home 内容
  - 客人通过旧链接看到的是当时的版本，不受后续修改影响
  - 原因：菜单、成员等信息可能因客人而异，需要固定
- **快照存储策略**：引用 + 差异模式
  - 照片不复制，只存引用 ID（避免存储膨胀）
  - 涂鸦和文字存版本差异
  - 照片被替换时不真正删除，标记为"归档"，旧快照引用仍有效
  - 归档照片定期清理（无引用的归档照片 90 天后清除）
- **新链接**：编辑后重新分享会生成新链接，包含最新内容

### 11.9 画板系统（两种画板）

#### 静态画板（直接在照片上操作）
- **入口**：每个房间卡的照片编辑区
- **工具**：画笔 + 橡皮擦 + 撤销 + 颜色选择器 + Home来画
- **产出**：绘画直接叠加在照片上，纯静态

#### 贴纸画板（独立小画板）
- **入口**：贴纸创建按钮
- **工具**：画笔 + 橡皮擦 + 撤销 + 颜色选择器 + Home来画
- **产出**：绘画完成后可设为贴纸 → 选择动效 → 存入贴纸库

#### Home来画（AI 辅助绘画，两种画板通用）
- **纯语言模式**：用户口头描述（如"帮我画一只趴在沙发上的猫"）→ AI 按当前材质调性生成对应风格的简笔画
- **画+语言模式**：用户先手绘草图 → 输入语言描述 → AI 以草图为参考 + 语言指令重绘优化
- 生成结果直接展现在画板上，用户可选接受或重新生成

#### 颜色规则
- 手动绘画：RGB 色相任选
- AI 生成（Home来画）：遵循材质调性的颜色规则（绒毛/石质/木质单色，叶脉/漫画/西幻彩色）

#### 扩充风格入口
- 调性选择器默认展示 4 种日常材质
- 底部设"更多风格"入口，展开可发现西幻和漫画

## 十三、风险与挑战

| 风险 | 应对 |
|------|------|
| 🎬 **一笔画动画质量** | M1 第一项任务，需找专业动画师，这是产品第一印象 |
| 🧙 **AI 涂鸦质量不稳定** | 预设 prompt 库 + Few-shot + 用户可重生成 |
| 🔗 **涂鸦与照片对齐** | M2 第一周做技术验证（spike），确认 Flux+ControlNet 叠加效果后再全面开发 |
| 💾 **快照数据复杂度** | M1 数据库设计阶段提前规划版本化策略（引用+差异+软删除） |
| 🤖 **引导师状态管理** | M1 设计阶段产出引导师状态机（idle/generating/presenting/dismissed） |
| 📱 **移动端绘画体验** | MVP 桌面端优先，移动端手动绘画降级为"仅选贴纸" |
| 💸 **AI 成本高** | 文字润色用 Gemini 2.5 Flash（~$0.15/1M token）+ 涂鸦用 Flux Schnell（~$0.003/张）+ AI 先分析感受性词汇只推荐 1-2 个版本 |
| 🐢 **3 分钟出作品难达成** | 引导师 + 模板 + 必填项最小化 |
| 🎨 **审美下限无法保证** | 材质调性锁定视觉风格（不可更改）+ 画板系统 + AI 美化 |
| 🔐 **真实住址泄露** | 默认不展示 + 链接私密 |
| 📉 **使用频次低** | 足迹地 + 叠层留言 + 季节活动 |
| ⚡ **多 AI Provider 故障** | Vercel AI SDK 统一接口 + 自动 fallback + 超时重试机制 |

## 十四、明确不做的事情

为了保持 vibe 和"小巧精致"，**以下功能明确不做**：

- ❌ **以盈利为导向的设计**（不接广告、不开会员、不卖模板；但用户可自由使用含商业用途）
- ❌ **创意工坊 V1**（可动贴纸独立创作）
- ❌ **微信小程序 V1**（V2 再说）
- ❌ **App**（V3 再说）
- ❌ **邻里 / 社区 / 发现页**
- ❌ **等级 / 积分 / 排行榜**
- ❌ **电商导流 / 真实家具**
- ❌ **复杂 AI 玩法**（自定义 prompt / 训练专属）

## 十六、产品 vibe 守则

> 这是 home 的"心法"，所有决策都问自己：
> **"这让 home 更像作品，还是更像工具？"**
>
> **品牌调性**：使用 Home 本身就是一种态度——
> 意味着主人对自己的生活和空间有着丰富的创造力与热爱。
> 无论私人住宅还是商业空间，这份表达是一致的。

- ✅ 保留**手绘的不完美**
- ✅ 优先**温暖**而不是**效率**
- ✅ 鼓励**表达**而不是**展示**
- ✅ 拒绝**机制化**（无积分、无等级、无排行榜）
- ✅ 慢节奏（动效慢、引导慢、用户有思考空间）
